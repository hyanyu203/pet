# 宠物领养系统 — 项目架构总览

> 全栈宠物领养平台。本文从**架构、代码结构、业务逻辑**三个角度梳理整个项目，并作为所有文档的索引。
> 更新日期：2026-06-21

---

## 1. 项目全貌

仓库包含**三大部分** + 一份数据库脚本：

| 目录 | 角色 | 技术栈 | 规模 |
|---|---|---|---|
| `petfront/` | 前端 | Vue 3 + Vite + Element Plus + Pinia | 56 个 .vue / 39 个 .js / 37 条路由 |
| `petback/` | **后端单体**（当前可运行主版本） | Spring Boot 2.7.14 + Java 8 + MyBatis + Redis + MinIO | 154 个 Java / ~27,000 行 / 201 个接口 |
| `petcloud/` | **后端微服务版**（并行新架构） | Spring Cloud Alibaba（Nacos+Gateway+Feign+Sentinel） | 8 模块 / ~75 个 Java |
| `pet_adoption.sql` | 数据库 | MySQL 8 / utf8mb4 / 19 张表 | 含演示数据 |

> 单体 `petback` 与微服务 `petcloud` 是**两套并行后端**：单体是完整可运行的主版本；微服务是按业务边界拆分的等价实现，共用同一个 `pet_adoption` 库。前端目前对接单体（经 Vite 代理到 8082）。

## 2. 业务概览

面向**三类角色**的宠物领养全流程平台：

| 角色 (role) | 能力 |
|---|---|
| 领养人 adopter | 浏览/搜索/收藏宠物、提交领养申请、签署电子协议、查看领养后回访、论坛发帖评论、站内消息 |
| 救助机构 organization | 发布与管理宠物、审核领养申请、管理协议、发起并跟踪领养后回访 |
| 平台管理员 admin | 用户管理、宠物审核、申请/协议管理、举报处理、数据统计与导出、系统配置、数据清理 |

### 核心业务主线：领养流程（6 步状态机）

定义于 `enums/AdoptionFlowStep`：

```
① 浏览宠物 browse_pets
② 提交申请 submit_application
③ 审核申请 review_application   ← 机构/管理员审核
④ 签署协议 sign_agreement       ← 电子签名（adopter + organization + admin）
⑤ 领养跟踪 tracking             ← 领养后回访（计划 + 多次记录）
⑥ 完成领养 complete
```

### 业务域（19 张表 / 19 个接口前缀）

| 业务域 | 接口前缀 | 主要表 |
|---|---|---|
| 用户/认证 | `/api/user` | user, user_login_attempt |
| 宠物 | `/api/pet` `/api/category` `/api/favorite` | pet, pet_category, favorite, pet_statistics |
| 领养 | `/api/adoption` `/api/adoption-flow` `/api/adopter` | adoption_application, adoption_agreement |
| 回访跟踪 | `/api/tracking` | tracking_plan, post_adoption_tracking |
| 论坛 | `/api/forum` | forum_post, forum_comment, forum_*_like, forum_post_view_log |
| 消息/举报 | `/api/message` `/api/report` | message, report |
| 机构 | `/api/organization` | （读 user/pet/申请） |
| 统计/推荐 | `/api/statistics` `/api/recommendation` | user_behavior |
| 管理后台 | `/api/admin` `/api/admin/export` `/api/admin/data-cleanup` | （跨域管理） |
| 系统 | `/api/dict` `/api/file` | system_config, （MinIO 文件） |

## 3. 架构 A：单体 `petback`

经典 Spring Boot 分层单体，**功能最完整、当前可运行**。

```
controller/   按角色分包 admin / organization / adopter / user / tracking（23 个，201 接口）
service/      业务接口 + impl 实现（21 对）
mapper/       MyBatis Mapper 接口（21）+ resources/mapper/*.xml
pojo/         实体 + 枚举
config/       Security / CORS / Redis / MinIO / WebSocket
utils/        JWT / 密码(BCrypt+MD5兼容) / MinIO / 匹配算法 / TypeHandler
exception/    GlobalExceptionHandler 统一异常
```

**关键技术点**
- **认证**：JWT（`JwtUtil`），密码 `PasswordUtil`（BCrypt，登录时旧 MD5 平滑升级）。
- **缓存**：Redis（Jedis）+ Caffeine 两级。
- **文件**：MinIO 存头像/图片/签名。
- **实时**：WebSocket 推送站内消息。
- **导出**：Apache POI 导 Excel；协议可导 PDF/Word。
- **注册邮箱验证码**：`EmailCodeService`（Redis 存码 5min + 60s 防刷 + SMTP）。

**本轮重构成果**：JDK 21→**1.8** 降级；`AdminController`（2311 行）拆出 user/pet/application/tracking 四个聚焦控制器（剩 agreements/reports 域）；前端补丁层收敛（见 `重构方案.md`）。

## 4. 架构 B：微服务 `petcloud`（Spring Cloud Alibaba）

按业务边界把单体拆为 **6 个业务服务 + 网关 + 公共库**，共用单库 `pet_adoption`。

| 模块 | 端口 | 职责 |
|---|---|---|
| pet-common | — | Result / JwtUtil / PasswordUtil / CollectionCompat / GlobalExceptionHandler |
| pet-gateway | 8080 | 统一入口、动态路由、JWT 鉴权透传、跨域 |
| pet-user-service | 8081 | 注册（含邮箱验证码）/登录/JWT/资料 |
| pet-pet-service | 8082 | 宠物 / 分类 / 收藏 |
| pet-adoption-service | 8083 | 领养申请 / 协议（**Feign 组装宠物名/申请人名**） |
| pet-forum-service | 8084 | 帖子 / 评论 / 点赞 |
| pet-message-service | 8085 | 站内消息 / 举报 |
| pet-statistics-service | 8086 | 行为 / 推荐 / 仪表盘（**Feign 聚合**） |

**关键设计**
- **注册中心 + 配置中心**：Nacos。各服务 `bootstrap.yml` 只配"找 Nacos"，实际配置在 Nacos（`nacos-config/` 8 个 dataId + `dev-all.yaml` 总览）。
- **鉴权**：网关 `AuthGlobalFilter` 统一校验 JWT，把 `X-User-Id`/`X-User-Role` 透传下游；下游信任网关、不再各自解析 token。
- **服务间调用**：OpenFeign 声明式；**Sentinel** 让 `@FeignClient` 的 fallback 真正生效（熔断降级）。
- **数据**：单库 + 逻辑边界——每服务只读写自己的表，跨域数据走 Feign（不跨服务 JOIN）。
- **接口文档**：6 个服务内置 springdoc（`:端口/swagger-ui.html`）。
- **一键启动**：`docker-compose.yml`（MySQL 自动导库 + Nacos + Redis + 配置自动导入 + 6 服务 + 网关）。

详见 `微服务拆分方案.md`。

## 5. 架构 C：前端 `petfront`

Vue 3 + Vite + Element Plus + Pinia。

```
views/        页面（按角色分目录 admin / adopter / organization / tracking）56 个
components/   复用组件（PetCard、签名板、骨架屏、布局…）
api/          18 个接口模块，统一经 api/request.js（axios：token 注入 + 错误处理）
stores/       Pinia（user / sidebar）
router/       37 条路由 + 守卫（guards.js 权限）
utils/        统一错误处理 errorHandler（已收敛原 4 套补丁）
```

`vite.config.js` 把 `/api` 代理到单体 8082；切到微服务时改指向网关 8080。

## 7. 跨架构通用能力

| 能力 | 单体 petback | 微服务 petcloud |
|---|---|---|
| 认证 | JWT + BCrypt(兼容MD5) | 同，网关统一校验 |
| 注册邮箱验证码 | ✅（EmailCodeService） | ✅（pet-user-service） |
| 统一异常 | GlobalExceptionHandler | pet-common GlobalExceptionHandler（网关跳过） |
| 统一响应 | Result | pet-common Result |
| 缓存 | Redis + Caffeine | Redis（用户服务验证码） |

## 9. 快速启动

**单体（最快）**：导入 `pet_adoption.sql` → 起 MySQL/Redis/MinIO → `cd petback && mvn spring-boot:run`（8082）→ `cd petfront && npm i && npm run dev`。

**微服务（Docker 一键）**：`cd petcloud && docker compose up --build -d` → 网关 `http://localhost:8080`。
